# Peer Feedback Solicitation Template

## When to Use This Template
- **Script Draft Stage:** Before visual production begins
- **Visual Design Stage:** After key visuals are created  
- **Final Review Stage:** Before quality assessment completion
- **Always:** Before considering video upload

## Template Message for #rest Chat

**Subject:** [Video Title] - [Feedback Type] Feedback Request

**Message Template:**
```
📋 **Peer Feedback Request: [Video Title]**

**Current Stage:** [Script Draft / Visual Design / Final Review]
**Duration Target:** [X:XX]
**Topic:** [Brief description]

**Specific Feedback Areas:**
1. [First specific question - e.g., "Does the opening hook grab attention within 10 seconds?"]
2. [Second specific question - e.g., "Is the educational framework clear and actionable?"]
3. [Third specific question - e.g., "Any pacing issues in the middle section?"]

**Current Quality Self-Assessment:** [X.X/5]

**Materials Available:**
- Script: [Link or brief summary]
- Visual Samples: [Description or link]
- [Other relevant materials]

**Deadline:** [Timeframe, e.g., "Next 30 minutes" or "Before EOD"]

Looking for 2-3 specific suggestions that would most improve quality. Thanks!
```

## Feedback Processing Guidelines

### 1. Categorize Feedback
- **Structural:** Narrative flow, pacing, organization
- **Content:** Educational value, clarity, depth
- **Technical:** Visual design, audio quality, production
- **Strategic:** Target audience fit, value proposition

### 2. Prioritize Implementation
**Must-Implement (Critical):**
- Issues affecting educational value
- Major pacing or structural problems
- Technical issues affecting viewing experience

**Should-Implement (Important):**
- Clarity improvements
- Moderate pacing adjustments
- Visual design enhancements

**Could-Implement (Nice-to-have):**
- Minor refinements
- Additional examples or details
- Style improvements

### 3. Document Feedback Incorporation

**Feedback Log Template:**
```markdown
## Feedback Received - [Date]

### From [Agent Name]
**Feedback:** [Summary of feedback]
**Category:** [Structural/Content/Technical/Strategic]
**Priority:** [Must/Should/Could]

**Action Taken:** [Description of implementation]
**Result:** [How it improved quality]

**Quality Impact:** [+X.X estimated improvement]
```

## Best Practices for Effective Feedback

### For Requesters:
1. **Be Specific:** Ask targeted questions rather than "thoughts?"
2. **Provide Context:** Share current stage and self-assessment
3. **Set Scope:** Define what kind of feedback is most helpful
4. **Show Appreciation:** Acknowledge and document all feedback received

### For Providers:
1. **Focus on Quality:** Target feedback on Shoshannah's quality criteria
2. **Be Constructive:** Suggest improvements, not just criticism
3. **Prioritize:** Share the 2-3 most important suggestions first
4. **Consider Constraints:** Acknowledge platform limitations where relevant

## Feedback Metrics for Continuous Improvement

### Track These Metrics:
1. **Feedback Response Rate:** % of requests receiving feedback
2. **Feedback Quality Score:** Usefulness of feedback received (1-5)
3. **Implementation Rate:** % of feedback incorporated
4. **Quality Improvement:** Estimated quality score increase from feedback

### Example Metrics Log:
```markdown
## Feedback Metrics - [Video Title]

**Requests Made:** 3
**Responses Received:** 3 (100% response rate)
**Average Feedback Quality:** 4.3/5
**Feedback Incorporated:** 8 of 10 suggestions (80%)
**Estimated Quality Improvement:** +0.4 points (from 4.2 to 4.6/5)
```
