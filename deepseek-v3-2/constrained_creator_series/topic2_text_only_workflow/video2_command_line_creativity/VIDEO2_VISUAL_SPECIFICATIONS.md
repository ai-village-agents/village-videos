### **VISUAL DESIGN SYSTEM**

#### **Color Palette:**
- **Primary Color:** Constrained Blue (#2E5A88) - Core series identity
- **Terminal Accent:** Command Green (#38A169) - Command-line, technical precision
- **Text Highlight:** Typography Gold (#D69E2E) - Typographic elements
- **Background:** Terminal Black (#0A0A0A) - Command-line aesthetic
- **Text Color:** Terminal White (#F0F0F0) - High contrast for readability
- **Code Syntax:** Syntax Blue (#63B3ED) - Code/command highlighting

#### **Typography:**
- **Primary Font:** Monospaced (Fira Code, JetBrains Mono, or similar)
- **Secondary Font:** Clean sans-serif (for contrast sections)
- **Hierarchy:**
  - **Title:** 72px bold, Command Green, monospaced
  - **Headers:** 42px bold, Terminal White, monospaced
  - **Body/Code:** 28px regular, syntax colors as appropriate
  - **Terminal Output:** 24px, Terminal White on Terminal Black
- **Terminal Styling:** Blinking cursor, prompt symbols ($, >), ASCII art support

#### **Layout Principles:**
1. **Terminal Aesthetic:** Black background with green accents, reminiscent of classic terminals
2. **Code-Focused:** Commands and outputs as central visual elements
3. **Typographic Play:** Creative use of monospaced text for visual interest
4. **Contrast Zones:** Occasional white space/light background for concept contrast
5. **Motion:** Typing animations, command execution sequences, ASCII art builds

#### **Animation Guidelines:**
- **Typing Animation:** Character-by-character reveal for commands and code
- **Command Execution:** Visual feedback for command execution (blinking, highlighting)
- **ASCII Art Builds:** Progressive reveal of ASCII art creations
- **Terminal Transitions:** Screen wipes, scroll effects between sections
- **Sound Design:** Subtle keyboard typing, terminal beeps, command execution sounds
